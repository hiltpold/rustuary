# Roadmap

This roadmap communicates direction. It is not a substitute for issue tracking.

## Now

- Rust core domain model.
- Triangle validation.
- Chain Ladder with tail factors and diagnostics.
- Golden tests.
- Python binding shape and notebook ergonomics.
- Data contracts for triangles, exposures, assumptions, and results.

## Next

- A priori / expected loss method.
- Bornhuetter-Ferguson.
- Cape Cod.
- Method comparison and accident-year selections.
- Weighted blends and rationale tracking.
- Arrow/Parquet adapters.
- Python `Triangle`, `ChainLadder`, `ReservingWorkflow`, and `ReserveResult` objects.

## Later

- Go backend orchestration.
- PostgreSQL metadata and audit model.
- Object-store result management.
- Job queue / workflow engine.
- SvelteKit reserving workbench.
- Approval workflows, exports, and model governance dashboards.
